# About CiteVyn

## Overview

CiteVyn gives you cited, checkable answers about AI developer tools — Claude
(API), Claude Code, OpenAI Codex, and Google Gemini — drawn straight from the
makers' official documentation. Every claim links to the exact source page so
you can verify it, and when the docs do not support an answer CiteVyn says it
does not know instead of guessing. This page is the canonical description of
CiteVyn the product, written so questions about CiteVyn itself are answered from
the index like any other question.

## CiteVyn Pro and membership

CiteVyn Pro is not live yet. CiteVyn is currently an MVP demo and everything
here is free to try. Pro is planned to add higher rate limits, exact lookups,
saved history, and shareable answers. For now there is no paid tier and no
billing — you can ask anything about Claude, Claude Code, Codex, or Gemini at no
cost.

## Coverage

CiteVyn covers four products today: Claude (API), Claude Code, OpenAI Codex, and
Google Gemini — using their official documentation only. Support for additional
tools such as ChatGPT and Cursor is on the roadmap, not in the MVP. CiteVyn does
not answer from private or user-supplied documents in the MVP; private-source
ingestion, SSO, and tenant isolation are part of the planned Enterprise
offering.

## Accuracy and trust

CiteVyn is designed not to hallucinate. Every answer is grounded in indexed
official documentation and links to the exact source page so you can confirm it.
If the indexed docs do not support an answer, CiteVyn refuses rather than
guessing. This grounding-and-citation approach is what separates CiteVyn from a
general chatbot that may state things that are not in any source.

## Freshness and the index

CiteVyn serves from the last known-good index, so a failed re-index never
corrupts what is already live. The corpus is a curated snapshot of the official
docs rather than a live crawl. Scheduled source refresh — keeping the index
continuously up to date — is a planned Enterprise feature.
